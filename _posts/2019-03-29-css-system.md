개발 과정에서 스타일 작업을 무엇보다 싫어하는가? 스타일 작업할 때 몇 시간 동안 공들여 만들 스타일이 깨질까 봐 두려웠던 적이 있는가? 그런 두려움을 없애줄 함수형 CSS에 대해 알아보자.

이 글에서는 함수형 CSS가 무엇인지 설명하고, 기존 CSS와 비교하고, 함수형 CSS를 사용하여 스타일을 단순화시키는 방법에 대해 설명하고자 한다.나는 CSS 쓰는 것을 싫어했다! 어디서부터 시작해야 될지도 몰랐다. CSS파일을 어떻게 구조화하는지, 언제 클래스를 만드는 게 적합한지, 혹은.. 어떻게 `div`는 세로로 가운데 정렬하는지?

이런 것들은 정기적으로 나타나던 질문들이었다. 나는 스타일에 조금의 작은 변경사항이라도 주는 게 두려웠다. 왜냐면 원하는 결과를 얻으려면 너무 많은 시간이 소요됐기 때문이고 이 모든 것이 나에게는 도무지 이해가 가지 않았다.

Angular Material, Ionic, Bootstrap과 같은 라이브러리를 발견했을 때 약간은 안심할 수 있었다. 덕분에 레이아웃 스타일, 버튼 스타일, div정렬에 대한 걱정보다 기능적인 부분을 개발하는데 집중할 수 있었다.

모든 것이 한동안 장밋빛이었지만, 이상한 기분이 들기 시작했다. 사용할수록 이런 라이브러리들이 굉장히 제한적이라는 걸 알게 되었다. 이러한 라이브러리의 개발자들은 나 같은 유저의 앱의 스타일과 모습을 거의 다 정해놓은 것과 다름없었고  그 후에 유저들이 할 수 있는 것은 많이 없었다.

게다가, 좋은 라이브러리는 가장 인기가 많기 때문에 대부분의 앱이 비슷한 모양과 느낌을 같게 된다. 모든 것이 비슷한 스타일에 속하게 된다.

그렇다고 오해하지는 말자. 그런 컴포넌트 라이브러리는 제품과 상관없는 많은 작업을 처리하기 때문에, MVP작업을 할 때는 정말 훌륭하다. 하지만 MVP를 벗어난 다음에는? Bootstrap을 죄다 뜯어내고 커스텀 CSS로  대체할 수 있겠는가?

**NOPE.**

함수형 CSS는 스타일 문제에 대해 완전히 새로운 시각을 제시한다. 앱이 멋지게 보이기를 원하지만, 다음 3주를 div를 중앙 정렬하는데 다 보내고 싶진 않다. (나만 div를 중앙 정렬하는데 매달렸나?)

## 함수형 CSS가 도대체 무엇인가?

함수형 (functional)의 의미는 다음과 같다.

> **특별한 활동, 목적 또는 업무를 수행하거나 가지고 있는 것 - 무언가가 작동하는 방식과 연관돼서.**

함수는 한 가지 작업을 수행해야 하며 동일한 입력이 주어지면 동일한 아웃풋을 생성해야 한다.

_매번!_

CSS에서는, 클래스는 하나의 효과를 적용하고, 매번 동일한 효과를 내야 한다. 따라서 결과적으로 여러분의 CSS는 많은 시각적 효과를 위해 여러 가지의 굉장히 작은 부분들로 구성됩니다.

## 해결사, Tachyon

언제 저렇게 많은 함수형 CSS 블록을 작성하느냐고 생각할 수도 있겠다. 어떤 게 필요한 줄 어떻게 알 수 있을까?

좋은 소식은, 직접 작성할 필요하 없다는 것이다. 이미 다른 이들이 우리를 위해 작성해 놓았고 필요한 거의 모든 클래스들이 포함된 라이브러리들이 많이 나와 있다. 라이브러리들은 모든 것을 다 포함하지는 않고 있지만, 대부분 사용하는 속성들은 다 담고 있다.

그중의 하나가 [Tachyon](http://tachyons.io/) 이다. 직접 몇 가지 프로젝트에서 사용했고, 실제로 굉장히 유용했다.

Tachyon이 일의 속도를 높이는데 얼마나 효과적이었는지 보여주기 위해 위에 설명한 Frello앱의 각 부분에서 얼마나 시간을 할애했는지 측정해 보았다.

* 앱 기능 구축
* 스타일링
  * 전형적인 CSS
  * 함수형 CSS (여기서는 Tachyon을 사용)

결과는 다음과 같다.

* 앱 기능 : 3시간 10분
* 전형적인 CSS 스타일링 : 3시간 44분
* 함수형 CSS 스타일링 : 1시간 59분

전형적인 CSS방식이 가장 오랜 시간이 걸렸다. 심지어 앱의 기능을 구축하는 것보다 더 오래 걸렸다.
아마도 기본 CSS를 작성하는 나의 능력에 대한 결과일지 모르겠지만, 하지만 어쨌든 함수형 CSS가 훨씬 더 빨랐다는 점은 분명하다.
Tachyons 클래스들을 컴포넌트에 작성하는 시간보다 전형적인 CSS를 작성하는데 거의 2배 가까운 시간이 들었다. 여기서 눈여겨보아야 할 차이점은 Tachyons를 사용하면 CSS를 많이 작성할 필요가 없다는 것이다. 처음부터 모든 함수형 CSS 클래스들을 직접 작성해야 한다면, 훨씬 더 많은 시간이 소요됐을 것이다.

## 왜 중요한가?

여기서 내가 옹호하고자 하는 것은, 다른 사람들이 만든 이와 같은 함수형 CSS 라이브러리를 효과적으로 이용하라는 것이다. 라이브러리는 디자인에 있어서 견고한 기초 위에 작성되었으며, 사람들이 많은 시간을 들여 가장 유용한 클래스들을 만들어 낸 결과이다.

라이브러리의 클래스만 유용한 것이 아니다. Tachyons가 가지고 있는 기본 디자인 원리(Fundamental design principle) 자체가 굉장히 유용하다. 모든 Tachyons의 간격과 사이즈 클래스들은 스케일에 비례한다.

![](/uploads/functional-css-03.png)

정확히 말하자면, Tachyons는 모든 크기와 간격에 대해 'rem'을 사용하여 앱의 루트 폰트 크기에 영향을 받는다. (relative)

예를 들어, 기본 루트 폰트 값이 16px이라고 한다면(바꾸지 않는 한) 4를 기반으로 한 스케일 시스템을 가지게 된다. 모든 간격과 사이즈는 4의 배수로 이루어진다.

애플과 구글은 이미 제품에 4를 기반으로 한 스케일 시스템(4-based scale system)을 사용하고 있고, 역시 디자인에 대해 뭘 좀 아는 거 같다.

>

일관된 간격 스케일을 사용하면 레이아웃을 보다 예측 가능하게 하고,  더 잘 맞고(fit), 더 잘 정렬됨으로써 비율을 통한 유지 관리성을 높일 수 있다.<br>
**Jina Anne, DesignBetter.Co**

디자인과 레이아웃에 리듬과 밸런스를 주기 때문에 매우 중요한 부분이다.

Tachyons의 타이포 스케일 시스템

스케일을 사용하면 마법과 같은 숫자들의 사용을 피할 수 있다. (CSS에서 한 번쯤은 사용해본, 갑자기 모든 것을 해결해 주는 그런 숫자 값 - 특정 고정 px을 주니까 모든 게 작동할 때..)

함수형 CSS를 사용하는 또 다른 장점은 탭 전환을 줄임으로써 개발 플로우의 속도를 높일 수 있다는 점이다. 특이 React와 같은 프레임 워크를 사용할 때 빛을 발한다.

함수형 CSS를 사용하면 단일 파일 안에서 모든 것을 해결할 수 있다. 위에서 설명한 것처럼 퀄리티 있는 아웃풋을 유지하면서도 컴포넌트 만드는데 들어가는 시간을 크게 단축할 수 있다.

## Tachyons Schmachyons

그런데 함수형 CSS는 스케일이 안되잖아.. 앱에서 특정 버튼을 전체적으로 업데이트하고 싶으면, 모든 버튼에 클래스를 바꿔야 되잖아!!

이건 가능한 걱정이다. 하지만 앱을 만드는 방식을 살펴보자. 앱에 걸쳐서 같은 버튼이 사용되고 있다면, 버튼을 컴포넌트로 만들어라. 그리고 버튼이 필요한 곳에 버튼 컴포넌트를 사용하고, 스타일을 업데이트하고 싶다면, 컴포넌트로 가서 클래스를 수정하면 된다!

Tachyons에는 많은 클래스들이 있고, 어떤 걸 사용해야 할지 알기 어려울 수 있다. 그리고 클래스가 뭘 하는지 조차 알기 힘들 수 있다.

'bn'은 뭐지? 'fw7'을 사용해야 하나?

[Tachyons의 도큐먼트](https://tachyons.io/) 는 유용하긴 하지만 탐색하기엔 어려울 수 있다. 특히나 특정 클래스의 기능을 찾을 때는 더욱 어렵다.

다행히도, [Tachyons TLDR](https://tachyons-tldr.now.sh/) 은 클래스 이름과 스케일 시스템을 포함해 많은 부분의 이해를 도와주는 유용한 사이트이다. 나에게는 Tachyons TLDR이 굉장히 유용했다.

함수형 CSS 라이브러리들에 대한 또 다른 공통적인 불만 중 하나는 이런 라이브러리들은 매우 독단적이고 제한된 클래스만 제공한다는 의견이다.

이런 것들 때문에 고민이라면, 딱 맞는 해결책이 있다.

Tachyons는 커스터마이징이 가능하고, tachyons-generator를 사용해 원하는 설정을 기반으로 스타일시트를 생성할 수 있다.

이외에도 다른 함수형 CSS 라이브러리들도 많이 있고 몇몇은 커스터마이징이 가능하니 살펴보기 바란다.

* Tailwind
* Basscss
* BuzzFeed's Solid

## 함수형 CSS의 세계로!

매번 다른 프로젝트에서 사용된 CSS 클래스와 거의 동일한 스타일을 작성하는데 시간을 허비하지 말고, 앱의 기능적인 면을 구축하는데 더 많은 에너지를 쓰자!

탭 전환을 줄여 작업에 소요되는 시간도 절약하고, 4년 과정을 거쳐 디자인 학위를 따지 않고서도 프로젝트에 견고한 디자인 원칙을 적용시킬 수 있을 것이다.

Tachyons이나 다른 함수형 CSS 라이브러리를 통해 스타일링을 단순화하자.
